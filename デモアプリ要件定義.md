# 1-bit LLM デモアプリ 要件定義書

> 作成日: 2026-04-17

---

## 1. アプリの目的

Bonsai-8B（1-bit LLM）と Gemma 4 E4B（通常のオンデバイスLLM）を同じ端末上で切り替えながらチャットし、**回答品質・応答速度・リソース消費の違いを体感できる** Androidアプリ。

Gemma 4 E4B選択時は画像入力・音声入力にも対応し、マルチモーダルの有無という1-bit LLMとの機能差も体感できるようにする。

---

## 2. 対象デバイス

| 項目 | 内容 |
|------|------|
| **検証端末** | Pixel 10 Pro |
| **OS** | Android 16 |
| **RAM** | 16 GB |
| **SoC** | Google Tensor G5 |

---

## 3. 搭載モデル

| モデル | タイプ | メモリ目安 | 形式 | 対応入力 |
|--------|--------|-----------|------|----------|
| **Bonsai-8B** | 1-bit LLM | ~1.15 GB | GGUF（Q1_0_g128） | テキストのみ |
| **Gemma 4 E4B** | オンデバイスLLM | ~2.5 GB | GGUF（Q4_K_M） | テキスト・画像・音声 |

---

## 5. 機能一覧

| ID | 機能 | 詳細 |
|----|------|------|
| F-1 | **チャット** | テキスト入力でAIとチャット。吹き出し形式で会話を表示 |
| F-2 | **モデル切り替え** | 左上のドロップダウンで Bonsai-8B / Gemma 4 E4B を切り替え。切り替え時にチャット履歴はリセット |
| F-3 | **応答情報の表示** | AI回答の吹き出し下部に、応答開始までの待ち時間（秒）と生成速度（tok/s）を表示 |
| F-4 | **リソースモニター** | 右上ボタン押下でオーバーレイ表示。モデルのメモリ使用量・端末全体のメモリ使用量・CPU使用率をリアルタイム表示 |
| F-5 | **ストリーミング出力** | AIの回答を1文字ずつ順番に表示（ChatGPTのような動き） |
| F-6 | **画像入力**（Gemma時のみ） | 入力欄左の画像ボタンからカメラ撮影またはギャラリー選択で画像を添付。テキストと一緒に送信 |
| F-7 | **音声入力**（Gemma時のみ） | 入力欄のマイクボタンで録音開始。停止すると音声をそのままGemma 4 E4Bに送信し、内容を認識して回答 |

## 6. 画面仕様の詳細

### 6.1 ヘッダー

| 要素 | 位置 | 動作 |
|------|------|------|
| モデル選択ドロップダウン | 左上 | タップでモデル一覧を表示。選択するとモデルを切り替え |
| リソースモニターボタン | 右上 | タップでリソースモニターをオーバーレイ表示/非表示 |

### 6.2 チャットエリア

| 要素 | 詳細 |
|------|------|
| ユーザーの発言 | 右寄せの吹き出し |
| AIの回答 | 左寄せの吹き出し。下部に応答時間と生成速度を小さく表示 |
| 画像付きメッセージ | 吹き出し内に画像サムネイル + テキストを表示（Gemma時） |
| 音声メッセージ | 吹き出し内に「🎤 音声入力」ラベル + 認識されたテキストの要約を表示（Gemma時） |
| スクロール | 新しいメッセージが下に追加。自動で最下部にスクロール |

### 6.3 入力欄（モデルによって切り替わる）

#### Bonsai-8B 選択時

| 要素 | 詳細 |
|------|------|
| テキスト入力フィールド | 1行表示。複数行入力時は自動で高さが伸びる（最大3行） |
| 送信ボタン | 右端。タップまたはキーボードのEnterで送信 |
| 推論中の状態 | 送信ボタンが無効化され、「生成中...」と表示 |

#### Gemma 4 E4B 選択時

| 要素 | 詳細 |
|------|------|
| 画像ボタン（🖼） | 入力フィールドの左。タップで「カメラで撮影」「ギャラリーから選択」を選べる |
| マイクボタン（🎤） | 画像ボタンの右隣。タップで録音開始。録音中は波形アニメーションと経過時間を表示 |
| テキスト入力フィールド | マイクボタンの右。Bonsai時と同じ動作 |
| 送信ボタン | 右端。画像が添付されている場合は画像+テキストを一緒に送信 |
| 画像プレビュー | 画像選択後、入力欄の上にサムネイル表示。✕ボタンで削除可能 |
| 推論中の状態 | 送信ボタンが無効化され、「生成中...」と表示 |

### 6.4 音声入力の動作フロー

```
マイクボタンをタップ
  → 録音開始（波形アニメーション + 経過時間を表示）
  → 「停止して送信」ボタンをタップ
  → 音声データをGemma 4 E4Bに直接送信
  → Gemmaが音声を認識し、内容に対して回答を生成
```

> Gemma 4 E4Bは音声エンコーダを内蔵しており、音声→テキスト変換（STT）とテキスト理解を1つのモデルで処理できる。Whisper等の外部STTモデルは不要。

### 6.5 リソースモニター（オーバーレイ）

| 表示項目 | 詳細 |
|---------|------|
| モデルのメモリ使用量 | 読み込んだモデルが使っているメモリ（GB） |
| 端末全体のメモリ使用量 | OS含む全体の使用メモリ / 合計メモリ |
| CPU使用率 | 端末全体のCPU使用率（%）。推論中に上昇する様子が見える |
| 更新頻度 | 1秒ごとに更新 |

## 7. 技術スタック

| 部品 | 技術 | 理由 |
|------|------|------|
| **UI** | Jetpack Compose | Android標準のモダンUIフレームワーク |
| **推論エンジン** | llama.cpp（Android NDK経由） | GGUF形式の両モデルに対応。CPU推論に最適化 |
| **Bonsai-8B対応** | prism-ml/llama.cpp フォーク | 1-bitカーネル対応のllama.cppフォーク |
| **画像入力** | llama.cppのマルチモーダル機能 | Gemma 4 E4BのVisionをllama.cpp経由で利用 |
| **音声入力** | llama.cppのオーディオ機能 | Gemma 4 E4Bの音声エンコーダ（40msフレーム、低遅延）をllama.cpp経由で利用 |
| **カメラ/ギャラリー** | Android CameraX / Photo Picker | カメラ撮影と画像選択のAndroid標準API |
| **マイク録音** | Android AudioRecord | 端末マイクからの音声取得 |
| **リソース取得** | Android Debug.MemoryInfo / /proc/stat | メモリとCPU使用率の取得 |
| **言語** | Kotlin + C++（JNI） | UI部分はKotlin、推論部分はC++ |

## 8. 動作に必要な環境

| 項目 | 要件 |
|------|------|
| **OS** | Android 14 以上 |
| **RAM** | 8 GB 以上（16 GB 推奨） |
| **ストレージ** | 4 GB 以上の空き（モデルファイル保存用） |
| **ネット接続** | 初回のモデルダウンロード時のみ必要。以降はオフラインで動作 |
| **権限** | カメラ、マイク、ストレージ（Gemma利用時） |

## 9. 制約・注意事項

- Bonsai-8Bはテキスト専用。Bonsai選択時は画像・音声ボタンを非表示にする
- Gemma 4 E4Bの音声入力はGemma内蔵の音声エンコーダで処理（外部STTモデル不要）
- 画像は送信前にリサイズ（長辺1024px以下）してメモリ消費を抑える
- 音声の最大録音時間は60秒とする
- モデル切り替え時はモデルのアンロード→ロードが発生するため数秒の待ち時間がある
- バックグラウンドに回った場合はモデルをアンロードしてメモリを解放する
